ü•û What are SwiftUI Stacks?

Stacks are layout containers that arrange views in specific patterns. Think of them as the organizational system for your UI - like arranging furniture in a room, but with code!

The Stack Family:

  • VStack: Vertical arrangement (top to bottom)
  • HStack: Horizontal arrangement (left to right)
  • ZStack: Depth arrangement (overlapping)
  • LazyVStack/LazyHStack: Performance-optimized versions
  • LazyVGrid/LazyHGrid: Grid arrangements
üé≠ Stacks are like organizing your closet: VStack is folding clothes vertically, HStack is hanging them horizontally, and ZStack is... well, throwing everything in a pile and hoping for the best! üòÖ

üìè VStack: The Vertical Virtuoso!

üìê VStack Fundamentals

VStack arranges its child views in a vertical line, from top to bottom. It's perfect for forms, lists, and any content that should flow vertically!

Key Properties:

  • alignment: .leading, .center, .trailing
  • spacing: Distance between views
  • content: The views to arrange

‚ö° Performance Tip:

VStack creates all child views immediately. For large lists or performance-critical content, consider LazyVStack instead!

üìö VStack is like organizing books on a shelf: Each book (view) sits on top of the previous one, and you can align them left, center, or right. Just don't let them fall over! üìñ

‚ÜîÔ∏è HStack: The Horizontal Hero!

üîÑ HStack Fundamentals

HStack arranges its child views in a horizontal line, from left to right. Perfect for navigation bars, button groups, and side-by-side content!

Key Properties:

  • alignment: .top, .center, .bottom, .firstTextBaseline
  • spacing: Distance between views
  • Spacer(): Pushes views apart

üé≠ ZStack: The Depth Master!

üåä ZStack Fundamentals

ZStack overlays its child views, creating depth and layering effects. Views are rendered from back to front, with later views appearing on top!

Key Properties:

  • alignment: Where views align (.center, .topLeading, etc.)
  • Z-index: Later views appear on top
  • Perfect for: Overlays, backgrounds, badges

‚ö° Z-Index Tips:

Views added later appear on top. Use .zIndex() modifier to explicitly control layering order!

üéÇ ZStack is like decorating a layer cake: You start with the base layer (background), add frosting (overlays), then sprinkles (details) on top. Each layer makes it more delicious! üç∞

‚ö° Lazy Stacks: The Performance Powerhouses!

üöÄ Lazy Stack Fundamentals

Lazy Stacks only create views when they're needed (when they become visible). Perfect for large datasets and scrolling performance!

The Lazy Family:

  • LazyVStack: Vertical lazy loading
  • LazyHStack: Horizontal lazy loading
  • Best for: Large lists, scrolling content, performance optimization

Advanced LazyVStack with Pinned Headers

                        AsyncImage(url: item.imageURL) { image in
                            image
                                .resizable()
                                .aspectRatio(contentMode: .fill)
                        } placeholder: {
                            Circle()
                                .fill(.gray.opacity(0.3))
                        }

                                .font(.headline)
                            Text(item.subtitle)

‚ö° When to Use Lazy Stacks:

  • Lists with 100+ items
  • Complex view hierarchies
  • Scrolling performance is critical
  • Memory usage needs to be optimized
ü¶• Lazy stacks are like a smart buffet: Instead of preparing all 1000 dishes at once (regular stack), they only cook what people are actually going to eat (visible views). Efficiency AND deliciousness! üçΩÔ∏è

üéØ Grid Stacks: The Layout Legends!

üèóÔ∏è Grid Stack Fundamentals

Grid Stacks arrange views in a grid pattern with flexible sizing and layout options. Perfect for photo galleries, app icons, and organized content!

The Grid Family:

  • LazyVGrid: Vertical scrolling grid
  • LazyHGrid: Horizontal scrolling grid
  • GridItem: Defines column/row behavior

Basic 2-Column Grid

let adaptiveColumns = [
    GridItem(.adaptive(minimum: 100))

Mixed-Size Dashboard Grid

let columns = [
    GridItem(.flexible(minimum: 100)),
    GridItem(.flexible(minimum: 100)),
    GridItem(.flexible(minimum: 100))
]

ScrollView {
    LazyVGrid(columns: columns, spacing: 16) {
        // Large featured card (spans 2 columns)
        VStack {
            Image(systemName: "chart.line.uptrend.xyaxis")
                .font(.system(size: 40))
                .foregroundColor(.green)
            Text("Revenue")
                .font(.headline)
            Text("$24,891")
                .font(.title2)
                .fontWeight(.bold)
        }
        .padding()
        .background(.green.opacity(0.1))
        .cornerRadius(16)
        .gridCellColumns(2) // Spans 2 columns
        
        // Regular cards
        ForEach(dashboardItems, id: \.id) { item in
            VStack(spacing: 8) {
                Image(systemName: item.icon)
                    .font(.system(size: 24))
                    .foregroundColor(item.color)
                Text(item.title)
                    .font(.caption)
                    .multilineTextAlignment(.center)
                Text(item.value)
                    .font(.headline)
                    .fontWeight(.semibold)
            }
            .padding()
            .background(.regularMaterial)
            .cornerRadius(12)
            .onTapGesture {
                selectedItem = item
            }
        }
    }
    .padding()
}

üí° GridItem Types:

  • .fixed(size): Exact size
  • .flexible(minimum, maximum): Flexible within bounds
  • .adaptive(minimum): Fit as many as possible
üß© Grid stacks are like organizing a jigsaw puzzle box: You arrange pieces (views) in perfect rows and columns, and when someone asks "Where's piece 247?" you can find it instantly! üîç
