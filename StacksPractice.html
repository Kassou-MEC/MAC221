Real iOS UIs.
You write the stacks.

Each challenge shows a fully rendered phone UI built from nested HStack, VStack, and ZStack. Reverse-engineer the SwiftUI code that produces it — no hints until you try.

⟷ HStack ↕ VStack ⧉ ZStack 🔀 All Nested
Reference

⟷ HStack

  • Left → right arrangement
  • alignment = vertical axis
  • Spacer() pushes items apart
  • Use for: rows, toolbars, icon+text

↕ VStack

  • Top → bottom arrangement
  • alignment = horizontal axis
  • Default outer screen container
  • Use for: columns, cards, forms

⧉ ZStack

  • Layers views back → front
  • First = back, last = front
  • alignment sets anchor point
  • Use for: overlays, badges, bg

🔀 Nesting Rule

  • Any stack can hold any stack
  • VStack(HStack(ZStack)) is normal
  • Sketch direction before coding
  • Extract to sub-views when > 4 deep
Strategy: For each challenge, identify the dominant direction (outer stack), then work inward. Every row inside a column = HStack inside VStack. Every overlay = ZStack. Build outside-in, not inside-out.

// Helper function for stat cards
func statCard(_ icon: String, _ value: String,
              _ label: String, _ color: Color) -> some View {
    VStack(alignment: .leading, spacing: 2) {
        Text(icon).font(.title3)
        Text(value).font(.title2).bold().foregroundColor(color)
        Text(label).font(.caption2).foregroundColor(.secondary)
    }
    .frame(maxWidth: .infinity, alignment: .leading)
    .padding(10)
    .background(Color(.secondarySystemBackground))
    .cornerRadius(12)
}
