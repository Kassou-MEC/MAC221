Welcome to Swift 🍎

Swift is Apple's powerful, modern programming language for iOS, macOS, watchOS, and tvOS. Designed to be safe, fast, and expressive — it's the foundation of every app you'll build in MAC221.

What is Swift?

🚀 Modern Language

Swift was introduced by Apple in 2014 as a replacement for Objective-C. It combines safety, performance, and modern syntax inspired by Python, Rust, and Haskell.

🔒 Safe by Design

Swift eliminates entire classes of common bugs. Optionals handle missing values, type safety prevents errors at compile-time, and memory management is automatic via ARC.

⚡ Blazing Fast

Swift compiles to native machine code. Apple benchmarks show it's faster than Python for many tasks, and comparable to C++ in performance-critical code.

🌍 Open Source

Swift is open source and runs on Linux and Windows too. You can use it for server-side Swift, command-line tools, and of course — iOS apps.

Section 2 · Variables & Constants

var vs let 📦

Every program stores data. Swift gives you two ways to store values: variables that can change, and constants that can't. Choosing the right one makes your code safer and clearer.

🎯 Learning Objectives

  • Declare variables with var and constants with let
  • Understand type inference vs explicit type annotations
  • Use string interpolation to embed values in strings
  • Apply naming best practices used in iOS development
  • Recognize and avoid beginner mistakes

var and let

var — Variable

The value can change after it's set. Use when data will update (score, temperature, username input).

let — Constant

The value cannot change once assigned. Use when data is fixed (pi, app name, API key). Prefer let by default!

VarsAndConstants.swift 
// VARIABLES — can be reassigned
var score = 0
score = 10        // ✅ OK!
score = score + 5  // ✅ Now score = 15

var userName = "Guest"
userName = "Maria"  // ✅ Updated

// CONSTANTS — cannot be reassigned
let appName = "MyApp"
let pi = 3.14159

// appName = "OtherApp"  // ❌ ERROR: Cannot assign to 'let'
// pi = 3.0              // ❌ ERROR: Constants are immutable

Type Inference vs Explicit Annotations

Swift is smart — it can figure out the type from the value you assign. This is called type inference. But you can also be explicit about the type.

TypeAnnotations.swift 
// TYPE INFERENCE — Swift figures it out
var age = 20           // Swift infers: Int
var gpa = 3.75         // Swift infers: Double
var isEnrolled = true  // Swift infers: Bool
var name = "Jordan"    // Swift infers: String

// EXPLICIT ANNOTATIONS — you specify the type
var credits: Int = 15
var tuition: Double = 5500.00
var major: String = "Computer Science"
var isFullTime: Bool = true

// Declaring without assigning (must annotate)
var finalGrade: String  // No value yet — must annotate
finalGrade = "A"         // Assign later

String Interpolation

String interpolation lets you embed values directly inside a string using \( ) syntax.

StringInterpolation.swift 
let student = "Aisha"
let course = "MAC221"
var grade = 95

print("Student: \(student)")
print("Course: \(course)")
print("Grade: \(grade)")
print("\(student) scored \(grade) in \(course)!")

// You can put expressions inside!
print("Next year grade: \(grade + 5)")
print("Passed: \(grade >= 60)")
Output

Course: MAC221
Grade: 95
Aisha scored 95 in MAC221!
Next year grade: 100
Passed: true

Arithmetic Operators

Operators.swift 
var a = 10

print(a + b)   // 13 — addition

print(a * b)   // 30 — multiplication

⚠️ Common Beginner Mistakes

❌ Mistake 1: Trying to change a let constant ▼

let is immutable. If you need to change a value, declare it with var. Swift will give you a compile-time error if you try to mutate a constant.

let score = 100
score = 90  // ❌ ERROR: Cannot assign to value: 'score' is a 'let' constant

var score2 = 100
score2 = 90 // ✅ Fine!
❌ Mistake 2: Mixing types without conversion ▼

Swift is strictly typed. You cannot add an Int and a Double directly — you must convert explicitly.

let age: Int = 20
let bonus: Double = 0.5

// let result = age + bonus  // ❌ ERROR: cannot add Int + Double

let result = Double(age) + bonus  // ✅ Convert Int to Double first
print(result)  // 20.5
❌ Mistake 3: Using string concatenation instead of interpolation ▼

In Swift, you can't add a non-String to a String with +. Use string interpolation instead — it's cleaner and more powerful.

let count = 5
// print("Items: " + count)          // ❌ ERROR!
print("Items: \(count)")            // ✅ Use interpolation
print("Items: " + String(count))  // ✅ Or convert first

Section 3 · Data Types

Swift Data Types 🗂️

Swift is a strongly-typed language with a rich set of built-in types. Understanding each type — and when to use it — is fundamental to writing correct Swift code.

🎯 Learning Objectives

  • Use Int, Double, Float, Bool, String, and Character correctly
  • Understand and safely unwrap Optionals
  • Work with Arrays, Dictionaries, Sets, and Tuples
  • Perform type conversions

Core Types — Click to Explore

Int stores whole numbers (no decimal). On modern Apple devices, Int is 64-bit, ranging from -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807.

Int.swift
let studentCount: Int = 856
var tapCount = 0          // inferred as Int
tapCount += 1

// Useful Int properties
print(Int.max)  // 9223372036854775807
print(Int.min)  // -9223372036854775808

// Arithmetic
let total = 10 + 5    // 15
let remain = 17 % 5   // 2 (modulo)

Double is 64-bit (15 decimal places of precision). Float is 32-bit (6 decimal places). Always prefer Double in Swift — it's the default for floating-point literals.

Double.swift
let price: Double = 4.99
let pi = 3.14159         // inferred as Double
let rating: Float = 4.5

// Calculation example
let radius = 5.0
let area = pi * radius * radius
print("Area: \(area)")   // Area: 78.53975

// Important: Int / Int = Int (truncated!)
print(7 / 2)             // 3, NOT 3.5!
print(7.0 / 2.0)         // 3.5 ✅

Bool holds only two values: true or false. Essential for conditions, toggles, and feature flags in apps.

Bool.swift
var isLoggedIn: Bool = false
var hasNotifications = true
let darkModeEnabled = false

// Toggle a boolean
isLoggedIn.toggle()   // Now true
isLoggedIn.toggle()   // Back to false

// From comparison
let score = 85
let passed = score >= 60   // true
let honors = score >= 90   // false

String is a sequence of characters. Swift Strings are Unicode-compliant, support emoji, and have powerful built-in methods.

String.swift
var greeting = "Hello"
let emoji = "Swift 🍎"

// String properties
print(greeting.count)         // 5
print(greeting.isEmpty)       // false
print(greeting.uppercased())  // HELLO
print(greeting.lowercased())  // hello

// Checking content
print(greeting.hasPrefix("He"))  // true
print(greeting.hasSuffix("lo"))  // true
print(greeting.contains("ell"))  // true

// Concatenation
greeting += ", World!"          // "Hello, World!"

// Multiline string
let bio = """
  Name: Alex
  Major: CS
  Year: Sophomore
  """

Character holds a single Unicode character. Must use double quotes, not single. Less commonly used directly — often encountered when iterating over Strings.

Character.swift
let initial: Character = "A"
let grade: Character = "B"
let emojiChar: Character = "🍎"

// Iterating String as Characters
let word = "Swift"
for char in word {
    print(char)   // S, w, i, f, t (one per line)
}

Optionals — Handling Missing Values

⚠️ Optionals are one of Swift's most important features. They force you to handle the case when a value might be nil (missing), preventing entire categories of crashes.
Optionals.swift
// Optional: a variable that might have a value or nil
var middleName: String? = nil   // ? makes it Optional
var nickname: String? = "Alex"

// --- WAY 1: if let (Safe Unwrapping) ---
if let name = nickname {
    print("Nickname: \(name)")  // Safe — only runs if not nil
} else {
    print("No nickname")
}

// --- WAY 2: guard let (Early Exit) ---
func greet(name: String?) {
    guard let n = name else {
        print("No name provided")
        return
    }
    print("Hello, \(n)!")
}

// --- WAY 3: Nil Coalescing (??) ---
let display = middleName ?? "N/A"   // "N/A" if nil
print("Middle: \(display)")

// --- WAY 4: Force Unwrap (use with caution!) ---
let forced = nickname!   // ⚠️ Crashes if nil! Only when SURE

Collections

Arrays are ordered collections of same-type values. Index starts at 0.

Arrays.swift
// Creating arrays
var courses: [String] = ["MAC221", "MAC280", "MAT115"]
var scores = [88, 92, 76, 95]  // inferred [Int]
var empty: [Int] = []           // empty array

// Accessing
print(courses[0])   // "MAC221"
print(courses.first) // Optional("MAC221")
print(courses.last)  // Optional("MAT115")

// Modifying
courses.append("MAC286")       // add to end
courses.insert("MAC101", at: 0) // insert at index
courses.remove(at: 2)          // remove index 2

// Properties
print(courses.count)     // number of items
print(courses.isEmpty)   // false

// Looping
for course in courses {
    print(course)
}

Dictionaries store key-value pairs. Fast lookup by key. Keys must be unique. Values are Optional when accessed.

Dictionary.swift
// Creating dictionaries
var grades: [String: Int] = [
    "Math": 90,
    "Swift": 95,
    "History": 82
]

// Accessing (returns Optional!)
print(grades["Swift"])     // Optional(95)
print(grades["Swift"]!)    // 95 (force unwrap)
print(grades["Art"] ?? 0) // 0 (default if missing)

// Adding / Updating
grades["English"] = 88
grades["Math"] = 93         // updates existing

// Removing
grades["History"] = nil

// Looping
for (subject, grade) in grades {
    print("\(subject): \(grade)")
}

Sets are unordered collections of unique values. Great for membership testing and removing duplicates. No guaranteed order.

Set.swift
// Creating sets
var favorites: Set<String> = ["Swift", "iOS", "Xcode"]
var nums: Set<Int> = [1, 2, 3, 2, 1]  // {1, 2, 3} — duplicates removed

// Operations
favorites.insert("SwiftUI")
favorites.remove("Xcode")
print(favorites.contains("Swift"))  // true

// Set math
let a: Set = [1, 2, 3]
let b: Set = [2, 3, 4]
print(a.union(b))        // {1,2,3,4}
print(a.intersection(b)) // {2,3}
print(a.subtracting(b))  // {1}

Tuples group multiple values of different types into a single compound value. Useful for returning multiple values from a function.

Tuple.swift
// Basic tuple
let location = (40.7128, -74.0060)   // (lat, long)
print(location.0)  // 40.7128
print(location.1)  // -74.0060

// Named tuple (much better!)
let student = (name: "Kim", grade: 92, passed: true)
print(student.name)   // Kim
print(student.grade)  // 92
print(student.passed) // true

// Tuple decomposition
let (n, g, p) = student
print("\(n) scored \(g)")  // Kim scored 92

Section 4 · Control Flow

Control Flow 🔀

Control flow lets your program make decisions and repeat actions. Swift offers powerful, readable syntax for branching with if/switch and looping with for/while.

🎯 Learning Objectives

  • Use if/else if/else for conditional branching
  • Use switch statements for pattern matching
  • Loop with for-in, while, and repeat-while
  • Use break and continue to control loop execution

if / else if / else

IfElse.swift
let score = 82

if score >= 90 {
    print("Grade: A")
} else if score >= 80 {
    print("Grade: B")   // ← This runs
} else if score >= 70 {
    print("Grade: C")
} else {
    print("Grade: F")
}

// Comparison operators
// ==  equal to        !=  not equal
// >   greater than    <   less than
// >=  greater/equal   <=  less/equal

// Logical operators
let age = 20
let hasID = true

if age >= 18 && hasID {           // AND: both true
    print("Access granted")
}

if age < 18 || !hasID {           // OR: either true
    print("Access denied")
}

switch — Pattern Matching

Swift's switch is far more powerful than other languages. No need for break — each case automatically ends. You can match ranges, tuples, and more.

Switch.swift
let day = "Monday"

switch day {
case "Saturday", "Sunday":
    print("Weekend! 🎉")
case "Monday":
    print("Start of the week")   // ← This runs
default:
    print("Weekday")
}

// Switch with ranges
let score = 85
switch score {
case 90...100: print("A")
case 80..<90:  print("B")    // ← This runs (85 is in 80..<90)
case 70..<80:  print("C")
default:         print("Below C")
}

// Switch must be exhaustive (cover all cases)
// Use 'default' if you can't list every case

Loops

Use for-in to iterate over a range, array, dictionary, or any sequence.

ForIn.swift
// Loop over a range
for i in 1...5 {
    print("Round \(i)")  // Round 1, 2, 3, 4, 5
}

// Half-open range (excludes 5)
for i in 0..<5 {
    print(i)            // 0, 1, 2, 3, 4
}

// Loop over array
let courses = ["MAC221", "MAC280", "MAT115"]
for course in courses {
    print("Enrolled: \(course)")
}

// With index
for (index, course) in courses.enumerated() {
    print("\(index + 1). \(course)")
}

// Ignore value with _
for _ in 1...3 {
    print("Ping!")  // prints 3 times
}

Use while when you don't know in advance how many times to loop. Checks condition before each iteration.

While.swift
// while loop — checks condition first
var lives = 3
var level = 1

while lives > 0 {
    print("Playing level \(level), lives: \(lives)")
    level += 1
    lives -= 1
}
print("Game over!")

// ⚠️ Make sure condition eventually becomes false!
// Infinite loop: while true { } — be careful

repeat-while executes the body at least once, then checks the condition. Like do-while in other languages.

RepeatWhile.swift
// repeat-while — runs body at least once
var attempts = 0

repeat {
    attempts += 1
    print("Attempt \(attempts)")
} while attempts < 3

// Output: Attempt 1, Attempt 2, Attempt 3

// Great for: asking user for input until valid

break exits a loop immediately. continue skips the current iteration and moves to the next.

BreakContinue.swift
// break — exits loop immediately
for i in 1...10 {
    if i == 5 {
        break  // stops at 5
    }
    print(i)  // prints 1, 2, 3, 4
}

// continue — skips current iteration
for i in 1...6 {
    if i % 2 == 0 {
        continue  // skip even numbers
    }
    print(i)      // prints 1, 3, 5
}

Functions & Closures 🔧

Functions are reusable blocks of code. Swift has expressive function syntax with argument labels, default parameters, and powerful closures that are used everywhere in iOS.

🎯 Learning Objectives

  • Declare and call functions with parameters and return values
  • Understand external argument labels and internal parameter names
  • Use default parameter values
  • Write functions that return multiple values with tuples
  • Understand closures as a concept

Basic Functions

Functions.swift
// No parameters, no return
func sayHello() {
    print("Hello, Swift!")
}
sayHello()   // Call the function

// With parameter
func greet(name: String) {
    print("Hello, \(name)!")
}
greet(name: "Aisha")  // Hello, Aisha!

// With return value
func add(a: Int, b: Int) -> Int {
    return a + b
}
let result = add(a: 5, b: 3)
print(result)  // 8

// Single-expression: implicit return
func square(n: Int) -> Int { n * n }
print(square(n: 4))  // 16

Argument Labels

Swift functions use external labels (for callers) and internal names (inside the function). This makes call sites read like English sentences.

ArgumentLabels.swift
// External label 'from', internal name 'city'
func travel(from departure: String, to destination: String) {
    print("Flying from \(departure) to \(destination)")
}
// Reads like English! 🎉
travel(from: "New York", to: "Casablanca")

// Suppress external label with _
func double(_ number: Int) -> Int {
    return number * 2
}
print(double(7))  // No label needed: double(7)

// iOS APIs use this pattern everywhere
// e.g., print("Hello") — no label
// e.g., array.insert("x", at: 0) — 'at' is the label

Default Parameters & Multiple Returns

DefaultParams.swift
// Default parameter values
func createProfile(name: String, role: String = "Student", age: Int = 18) {
    print("\(name) | \(role) | Age: \(age)")

func minMax(array: [Int]) -> (min: Int, max: Int) {

// Min: 60, Max: 95

    case 70..<80:  return "C"
    case 60..<70:  return "D"

    }

Closures — Functions as Values

💡 Closures are self-contained blocks of code that can be passed around. They're used constantly in iOS for callbacks, animations, and async code.
Closures.swift
// A closure stored in a variable
let greet = { (name: String) -> String in
    return "Hello, \(name)!"
}
print(greet("Swift"))  // Hello, Swift!

// Closures with array methods (VERY common in iOS)
let scores = [72, 88, 45, 95, 60]

// filter — keep only passing scores
let passing = scores.filter { score in score >= 60 }
print(passing)   // [72, 88, 95, 60]

// map — transform each element
let doubled = scores.map { $0 * 2 }
print(doubled)   // [144, 176, 90, 190, 120]

// sorted — custom sort
let sorted = scores.sorted { $0 > $1 }
print(sorted)    // [95, 88, 72, 60, 45] (descending)

Section 6 · Classes & Structures

Classes & Structures 🏗️

Classes and structs are the building blocks of Swift apps. Both define types with properties and methods — but they differ fundamentally in how they pass data around.

🎯 Learning Objectives

  • Define classes and structs with properties, methods, and initializers
  • Understand value types (struct) vs reference types (class)
  • Use mutating methods with structs
  • Choose between class and struct for the right situations
  • Apply OOP concepts to realistic iOS examples

Structures

Struct.swift
struct Student {
    // Stored properties
    var name: String
    var gpa: Double
    let studentID: Int   // constant property
    
    // Method
    func introduce() {
        print("Hi, I'm \(name) | GPA: \(gpa)")
    }
    
    // Mutating method (needed for structs!)
    mutating func updateGPA(_ newGPA: Double) {
        gpa = newGPA
    }
}

// Structs get a FREE memberwise initializer
var s1 = Student(name: "Aisha", gpa: 3.8, studentID: 1001)
s1.introduce()         // Hi, I'm Aisha | GPA: 3.8
s1.updateGPA(3.9)     // Must call with mutating
print(s1.gpa)           // 3.9

Classes

Class.swift
class AppUser {
    // Stored properties
    var username: String
    var loginCount: Int
    
    // Initializer (classes need it — no free memberwise!)
    init(username: String) {
        self.username = username
        self.loginCount = 0
    }
    
    // Method
    func login() {
        loginCount += 1
        print("\(username) logged in. Total: \(loginCount)")
    }
}

let user = AppUser(username: "aisha99")
user.login()   // aisha99 logged in. Total: 1
user.login()   // aisha99 logged in. Total: 2

// Inheritance (classes only!)
class AdminUser: AppUser {
    func deleteAccount(of target: String) {
        print("\(username) deleted \(target)'s account")
    }
}

Value Types vs Reference Types

⚠️ This is the most important difference between structs and classes. Structs are copied. Classes are shared.
ValueVsReference.swift
// STRUCT = Value Type (COPY)
struct Point { var x: Int; var y: Int }

var p1 = Point(x: 1, y: 2)
var p2 = p1       // COPY — p2 is independent
p2.x = 99
print(p1.x)       // 1 — unchanged! ✅
print(p2.x)       // 99

// CLASS = Reference Type (SHARE)
class Box { var value: Int = 0 }

var box1 = Box()
var box2 = box1    // SAME object! Not a copy
box2.value = 99
print(box1.value)  // 99 — also changed! ⚠️
print(box2.value)  // 99

Class vs Struct — When to Use?

Feature Class Struct
Type Reference (shared) Value (copied)
Inheritance ✅ Yes ❌ No
Deinitializer ✅ deinit ❌ No
Mutating keyword ❌ Not needed ✅ Required
Memberwise init ❌ Must write init ✅ Auto-generated
Performance Heap (slower alloc) Stack (faster)
SwiftUI Usage ObservableObject Views, Models (preferred!)
✅ Apple's recommendation: Start with a struct. Only switch to a class if you need inheritance, identity, or shared mutable state. In SwiftUI, nearly everything is a struct!

iOS App Example

TaskApp.swift
// A realistic iOS task app model
struct Task {
    let id: Int
    var title: String
    var isCompleted: Bool = false
    var priority: String = "Normal"
    
    mutating func complete() {
        isCompleted = true
        print("✅ '\(title)' completed!")
    }
    
    func summary() -> String {
        let status = isCompleted ? "✅" : "⏳"
        return "\(status) [\(priority)] \(title)"
    }
}

class TaskManager {
    var tasks: [Task] = []
    
    func add(_ task: Task) {
        tasks.append(task)
    }
    
    func showAll() {
        tasks.forEach { print($0.summary()) }
    }
}

// Usage
let manager = TaskManager()
manager.add(Task(id: 1, title: "Build login screen", priority: "High"))
manager.add(Task(id: 2, title: "Write unit tests"))
manager.showAll()

⚠️ Common Beginner Mistakes

1. Forgetting to unwrap optionals ▼

Always unwrap optionals with if let, guard let, or ?? before using them. Force unwrapping with ! crashes if the value is nil.

2. Missing mutating on struct methods ▼

Any struct method that modifies a property needs the mutating keyword. Classes don't need it.

3. Integer division truncation ▼

7 / 2 in Swift equals 3, not 3.5! Use Double values if you need decimal results: 7.0 / 2.0 = 3.5

4. Modifying a let constant ▼

Constants declared with let cannot be changed. If you get "Cannot assign to value" — switch to var.

5. Switch without default ▼

Switch must be exhaustive. If you can't cover every case, add default: break at the end.

6. Confusing class and struct copying behavior ▼

Structs are copied. Classes are shared. Assigning a class instance to a new variable gives you the same object, not a copy.
