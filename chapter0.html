Chapter 3

Optionals

Swift's Safety Feature
🎁 Think of Optionals as Gift Boxes

An optional is like a box that might contain a value or might be empty (nil). You must safely unwrap it to see what's inside!

Declaration
var userName: String?
var userAge: Int? = nil
var email: String? = "user@example.com"
Safe Unwrapping
if let name = userName
{ // Safely use name here
print("Hello, \(name)!")
} else {
print("No name provided")
}
Guard Statement
guard let name = userName else { print("No name!") return } // Use name safely here
Nil Coalescing
let displayName = userName ?? "Guest User" // Provides default if nil

Chapter 10

Protocols

Contracts for Types
📋 Think of Protocols as Contracts

Protocols define what methods and properties a type must have, without implementing them.

📝 Protocol Definition
protocol Drawable { var area: Double { get } func draw() } protocol Named { var name: String { get set } }
✅ Conformance
struct Circle: Drawable { let radius: Double var area: Double { return Double.pi * radius * radius } func draw() { print("Drawing circle") } }
🔧 Extensions
extension Drawable { func describe() { print("Area: \(area)") } } // All Drawable types get this method! let circle = Circle(radius: 5) circle.describe()
🏗️ Protocol Inheritance
protocol Vehicle { func start() } protocol Car: Vehicle { var doors: Int { get } } // Car protocol includes Vehicle requirements

Extensions

Adding Functionality to Existing Types
🔧 Extending Built-in Types
extension String { func isValidEmail() -> Bool { return self.contains("@") && self.contains(".") } var wordCount: Int { return self.components(separatedBy: .whitespaces).count } } // Usage let email = "user@example.com" print(email.isValidEmail()) // true
🏗️ Extending Custom Types
struct Temperature { var celsius: Double } extension Temperature { var fahrenheit: Double { return (celsius * 9/5) + 32 } init(fahrenheit: Double) { self.init(celsius: (fahrenheit - 32) * 5/9) } }

Error Handling

Managing Failures Gracefully
🚨 Defining Errors
enum ValidationError: Error { case tooShort(minimum: Int) case tooLong(maximum: Int) case invalidCharacters }
⚡ Throwing Functions
func validatePassword(_ password: String) throws { if password.count < 8 { throw ValidationError.tooShort(minimum: 8) } }
🛡️ do-catch
do { try validatePassword("abc") print("Password valid!") } catch ValidationError.tooShort(let min) { print("Too short! Need \(min) chars") } catch { print("Other error: \(error)") }
🤞 Optional Try
// try? - returns nil on error let result = try? validatePassword("abc") // try! - crashes on error (careful!) try! validatePassword("validPassword123")

Chapter 13

Closures

Functions as Values
🎁 Closures: Functions You Can Pass Around

Closures are self-contained blocks of functionality that can be passed as parameters. They're essential in SwiftUI for handling events and data transformation.

📝 Basic Syntax
// Full syntax let multiply: (Int, Int) -> Int = { (a: Int, b: Int) -> Int in return a * b } // Simplified let add = { (a: Int, b: Int) in a + b }
🎯 Higher-Order Functions
let numbers = [1, 2, 3, 4, 5] // map - transform each element let doubled = numbers.map { $0 * 2 } // filter - select elements let evens = numbers.filter { $0 % 2 == 0 } // reduce - combine into single value let sum = numbers.reduce(0) { $0 + $1 }
🎪 Trailing Closures
// Instead of this: numbers.map({ number in number * 2 }) // Write this (trailing closure): numbers.map { number in number * 2 } // Or even shorter: numbers.map { $0 * 2 }
📱 SwiftUI Usage
// Button with closure Button("Tap Me") { print("Button tapped!") } // List with closures List(items) { item in Text(item.name) }

Chapter 14

Access Control

Controlling Code Visibility
🌍 public
Accessible everywhere
public class User { public var name: String public init(name: String) { self.name = name } }
🏢 internal
Same module only (default)
// internal is the default class DatabaseManager { var connection: String func connect() { // Internal by default } }
📄 fileprivate
Same file only
class LoginManager { fileprivate var apiKey: String fileprivate func validateKey() { // Accessible within this file } }
🔒 private
Same declaration only
class SecureVault { private var password: String private func encrypt() { // Only this class can access } }
🎯 Best Practice: Start Private, Then Open Up

Begin with the most restrictive access level (private) and only make things more accessible as needed. This creates safer, more maintainable code.
