01-A

NavigationStack — Your First Stack

🟢 Level 1 — Absolute Beginner

The simplest possible NavigationStack. One screen with a title. No links yet — just the container itself.

ℹ️
What NavigationStack does It wraps your view in a navigation container. This is required before any NavigationLink will work. The .navigationTitle() modifier sets the large title at the top.
ContentView.swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        NavigationStack {
            Text("Hello, Navigation!")
                .font(.largeTitle)
                .navigationTitle("Home")
        }
    }
}
✅
What to observe in Xcode You'll see a large "Home" title at the top of the screen. The title shrinks when you scroll. That's .navigationBarTitleDisplayMode(.large) — the default.

🧪 Try It

  • Change the title to your name. What happens?
  • Add .navigationBarTitleDisplayMode(.inline) — what changes?
  • Replace Text with a Color.blue — the title still shows. Why?

NavigationStack — Multiple Links

🟢 Level 1 — Simple Navigation Links

Now let's add links. Each NavigationLink is a tappable row that pushes a new view onto the stack.

ContentView.swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        NavigationStack {
            List {
                NavigationLink("Go to Screen A", destination: ScreenA())
                NavigationLink("Go to Screen B", destination: ScreenB())
                NavigationLink("Go to Screen C", destination: ScreenC())
            }
            .navigationTitle("Main Menu")
        }
    }
}

// ── Destination Views ──────────────────────

    var body: some View {
        Text("You are on Screen A")
            .font(.title)
            .navigationTitle("Screen A")
    }
}

struct ScreenB: View {
    var body: some View {
        Text("You are on Screen B")
            .font(.title)
            .navigationTitle("Screen B")
    }
}

struct ScreenC: View {
    var body: some View {
        Text("You are on Screen C")
            .font(.title)
            .navigationTitle("Screen C")
    }
}

01-C

NavigationStack — Adding a Toolbar

🟢 Level 1 — Toolbar Buttons

Toolbars let you add buttons to the navigation bar. This is how you add "Add", "Edit", or "Done" buttons in real apps.

ContentView.swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        NavigationStack {
            Text("My Screen")
                .font(.largeTitle)
                .navigationTitle("Notes")
                .toolbar {
                    // Button on the TOP RIGHT
                    ToolbarItem(placement: .topBarTrailing) {
                        Button("Add") {
                            print("Add tapped!")
                        }
                    }
                    // Button on the TOP LEFT
                    ToolbarItem(placement: .topBarLeading) {
                        Button("Edit") {
                            print("Edit tapped!")
                        }
                    }
                }
        }
    }
}
// Note: .toolbar belongs on the VIEW INSIDE NavigationStack, not on NavigationStack itself.
⚠️
Common Mistake Do NOT put .toolbar on the NavigationStack. Put it on the view inside the stack (the Text, List, etc.). The stack just wraps everything.

🧪 Try It

  • Replace Button("Add") with a system icon button: Button(action: {}) { Image(systemName: "plus") }
  • Add a @State var count = 0 and make the "Add" button increment it. Display count in a Text.
  • Try placement: .bottomBar — where does the button appear?
01-D

NavigationStack — Passing Data to a View

🔵 Level 2 — Intermediate

Real apps pass data when navigating. Here we create a simple model and pass it to the destination view via its initializer.

ContentView.swift
import SwiftUI

// Step 1: Define a simple model
struct Fruit {
    let name: String
    let emoji: String
    let color: Color
}

// Step 2: Root view with a list of fruits
struct ContentView: View {
    let fruits = [
        Fruit(name: "Apple",  emoji: "🍎", color: .red),
        Fruit(name: "Banana", emoji: "🍌", color: .yellow),
        Fruit(name: "Grape",  emoji: "🍇", color: .purple)
    ]

    var body: some View {
        NavigationStack {
            List(fruits, id: \.name) { fruit in
                // Pass the entire fruit object to the detail view
                NavigationLink(destination: FruitDetailView(fruit: fruit)) {
                    Text("\(fruit.emoji) \(fruit.name)")
                        .font(.title3)
                }
            }
            .navigationTitle("Fruits")
        }
    }
}

// Step 3: Detail view that RECEIVES the data
struct FruitDetailView: View {
    let fruit: Fruit   // stored as a constant

    var body: some View {
        VStack(spacing: 20) {
            Text(fruit.emoji)
                .font(.system(size: 100))
            Text(fruit.name)
                .font(.largeTitle)
                .bold()
                .foregroundColor(fruit.color)
        }
        .navigationTitle(fruit.name)
        .navigationBarTitleDisplayMode(.inline)
    }
}

🧪 Try It

  • Add a calories: Int field to Fruit and display it in FruitDetailView.
  • Add a 4th fruit of your choice.
  • Make the background of FruitDetailView use fruit.color.opacity(0.1).
01-E

NavigationStack — Identifiable List

🔵 Level 2 — Intermediate

Using Identifiable with a list is the standard approach for iterating data models in SwiftUI.

ContentView.swift
import SwiftUI

// Conform to Identifiable — gives each item a unique id
struct Student: Identifiable {
    let id = UUID()   // auto-generated unique ID
    let name: String
    let grade: String
    let gpa: Double
}

struct ContentView: View {
    let students: [Student] = [
        Student(name: "Amina",   grade: "A", gpa: 3.9),
        Student(name: "Carlos",  grade: "B", gpa: 3.2),
        Student(name: "Diana",   grade: "A", gpa: 3.7),
        Student(name: "Elias",   grade: "B+", gpa: 3.5)
    ]

    var body: some View {
        NavigationStack {
            List(students) { student in
                NavigationLink(destination: StudentDetailView(student: student)) {
                    HStack {
                        Text(student.name)
                            .font(.headline)
                        Spacer()
                        Text(student.grade)
                            .foregroundColor(.secondary)
                    }
                }
            }
            .navigationTitle("Students")
        }
    }
}

struct StudentDetailView: View {
    let student: Student

    var body: some View {
        VStack(spacing: 16) {
            Image(systemName: "person.circle.fill")
                .font(.system(size: 80))
                .foregroundColor(.blue)
            Text(student.name)
                .font(.largeTitle).bold()
            Text("Grade: \(student.grade)")
                .font(.title2)
            Text("GPA: \(student.gpa, specifier: "%.1f")")
                .font(.title3)
                .foregroundColor(.secondary)
        }
        .navigationTitle(student.name)
        .navigationBarTitleDisplayMode(.inline)
    }
}

NavigationStack — Complete App Structure

🟠 Level 3 — Complex

A full multi-level navigation app: Categories → Items → Detail. Sections, custom rows, badges, and toolbar actions all working together.

ContentView.swift
import SwiftUI

// ── Models ─────────────────────────────────────────
struct Course: Identifiable {
    let id = UUID()
    let title: String
    let code: String
    let icon: String
    let topics: [Topic]
}

struct Topic: Identifiable {
    let id = UUID()
    let title: String
    let week: Int
    let description: String
    var isCompleted: Bool
}

// ── Sample Data ─────────────────────────────────────
let sampleCourses: [Course] = [
    Course(title: "iOS Development", code: "MAC221", icon: "iphone",
           topics: [
               Topic(title: "SwiftUI Basics", week: 1,
                     description: "Views, modifiers, layout system", isCompleted: true),
               Topic(title: "Navigation", week: 2,
                     description: "Stack, tabs, sheets", isCompleted: true),
               Topic(title: "State Management", week: 3,
                     description: "@State, @Binding, @Observable", isCompleted: false)
           ]),
    Course(title: "Data Structures", code: "MAC286", icon: "list.bullet.rectangle",
           topics: [
               Topic(title: "Arrays & Lists", week: 1,
                     description: "Linear data structures", isCompleted: true),
               Topic(title: "Trees", week: 4,
                     description: "Binary trees and traversal", isCompleted: false)
           ])
]

// ── Level 1: Course List ─────────────────────────────
struct ContentView: View {
    @State private var courses = sampleCourses

    var body: some View {
        NavigationStack {
            List(courses) { course in
                NavigationLink(destination: CourseView(course: course)) {
                    HStack(spacing: 14) {
                        Image(systemName: course.icon)
                            .font(.system(size: 28))
                            .foregroundColor(.blue)
                            .frame(width: 38)
                        VStack(alignment: .leading, spacing: 3) {
                            Text(course.title).font(.headline)
                            Text(course.code)
                                .font(.caption)
                                .foregroundColor(.secondary)
                        }
                        Spacer()
                        Text("\(course.topics.count)")
                            .font(.caption.bold())
                            .padding(.horizontal, 8)
                            .padding(.vertical, 4)
                            .background(Color.blue.opacity(0.12))
                            .foregroundColor(.blue)
                            .clipShape(Capsule())
                    }
                    .padding(.vertical, 4)
                }
            }
            .navigationTitle("My Courses")
            .toolbar {
                ToolbarItem(placement: .topBarTrailing) {
                    Button {  } label: {
                        Image(systemName: "plus.circle.fill")
                    }
                }
            }
        }
    }
}

// ── Level 2: Topics in a Course ──────────────────────
struct CourseView: View {
    let course: Course

    var completedCount: Int {
        course.topics.filter{ $0.isCompleted }.count
    }

    var body: some View {
        List(course.topics) { topic in
            NavigationLink(destination: TopicDetailView(topic: topic)) {
                HStack {
                    Image(systemName: topic.isCompleted
                        ? "checkmark.circle.fill"
                        : "circle")
                        .foregroundColor(topic.isCompleted ? .green : .secondary)
                    VStack(alignment: .leading, spacing: 2) {
                        Text(topic.title).font(.headline)
                        Text("Week \(topic.week)")
                            .font(.caption)
                            .foregroundColor(.secondary)
                    }
                }
            }
        }
        .navigationTitle(course.title)
        .navigationBarTitleDisplayMode(.inline)
        .safeAreaInset(edge: .bottom) {
            HStack {
                Text("\(completedCount) of \(course.topics.count) completed")
                    .font(.subheadline)
                    .foregroundColor(.secondary)
            }
            .padding()
            .frame(maxWidth: .infinity)
            .background(.ultraThinMaterial())
        }
    }
}

// ── Level 3: Topic Detail ────────────────────────────
struct TopicDetailView: View {
    let topic: Topic

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 20) {
                HStack {
                    Text("Week \(topic.week)")
                        .font(.caption)
                        .padding(.horizontal, 10)
                        .padding(.vertical, 5)
                        .background(Color.blue.opacity(0.1))
                        .cornerRadius(6)
                    Spacer()
                    Image(systemName: topic.isCompleted
                        ? "checkmark.seal.fill"
                        : "seal")
                        .foregroundColor(topic.isCompleted ? .green : .secondary)
                }
                Text(topic.description)
                    .font(.body)
                    .lineSpacing(6)
            }
            .padding()
        }
        .navigationTitle(topic.title)
        .navigationBarTitleDisplayMode(.inline)
    }
}

02-C

NavigationLink — Wrapping Any View

🟢 Level 1 — Custom Label

A NavigationLink can wrap any view as its label — a button, an image, even a whole card.

ContentView.swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        NavigationStack {
            VStack(spacing: 20) {

                // A NavigationLink wrapped around a styled button
                NavigationLink(destination: RedView()) {
                    Text("Go to Red Screen")
                        .font(.headline)
                        .frame(maxWidth: .infinity)
                        .padding()
                        .background(Color.red)
                        .foregroundColor(.white)
                        .cornerRadius(12)
                }

                NavigationLink(destination: GreenView()) {
                    Text("Go to Green Screen")
                        .font(.headline)
                        .frame(maxWidth: .infinity)
                        .padding()
                        .background(Color.green)
                        .foregroundColor(.white)
                        .cornerRadius(12)
                }
            }
            .padding()
            .buttonStyle(.plain) // removes default blue tint
            .navigationTitle("Colors")
        }
    }
}

struct RedView: View {
    var body: some View {
        Color.red.ignoresSafeArea()
            .navigationTitle("Red")
    }
}
struct GreenView: View {
    var body: some View {
        Color.green.ignoresSafeArea()
            .navigationTitle("Green")
    }
}
💡
Why .buttonStyle(.plain)? By default, SwiftUI tints all NavigationLink labels blue. Adding .buttonStyle(.plain) lets your custom colors show through unchanged.

02-E

NavigationLink — Value-Based (iOS 16+)

🟠 Level 3 — Advanced

Instead of embedding the destination inside each link, NavigationLink(value:) + .navigationDestination(for:) centralize all routing in one place. The model must be Hashable.

ContentView.swift
import SwiftUI

// Model must be Hashable for value-based navigation
struct Animal: Identifiable, Hashable {
    let id = UUID()
    let name: String
    let emoji: String
    let habitat: String
}

struct ContentView: View {
    let animals = [
        Animal(name: "Lion",     emoji: "🦁", habitat: "Savanna"),
        Animal(name: "Penguin", emoji: "🐧", habitat: "Antarctica"),
        Animal(name: "Dolphin", emoji: "🐬", habitat: "Ocean"),
        Animal(name: "Eagle",   emoji: "🦅", habitat: "Mountains")
    ]

    var body: some View {
        NavigationStack {
            List(animals) { animal in
                // Pass the VALUE, not the view
                NavigationLink(value: animal) {
                    Text("\(animal.emoji) \(animal.name)")
                        .font(.title3)
                }
            }
            .navigationTitle("Animals")
            // ONE place to handle ALL Animal navigations
            .navigationDestination(for: Animal.self) { animal in
                AnimalDetailView(animal: animal)
            }
        }
    }
}

struct AnimalDetailView: View {
    let animal: Animal

    var body: some View {
        VStack(spacing: 20) {
            Text(animal.emoji).font(.system(size: 100))
            Text(animal.name).font(.largeTitle).bold()
            Label("Habitat: \(animal.habitat)", systemImage: "mappin")
                .font(.title3)
                .foregroundColor(.secondary)
        }
        .navigationTitle(animal.name)
        .navigationBarTitleDisplayMode(.inline)
    }
}

Programmatic Nav — Bool Binding

🟢 Level 1 — Simple Bool

The simplest programmatic navigation: trigger navigation from a Button using a @State Bool. No user tapping a NavigationLink required.

ContentView.swift
import SwiftUI

struct ContentView: View {
    @State private var goToDetail = false

    var body: some View {
        NavigationStack {
            VStack(spacing: 30) {
                Text("Press the button to navigate")
                    .font(.title2)

                Button("Go to Detail Screen") {
                    goToDetail = true  // triggers navigation
                }
                .buttonStyle(.borderedProminent())

                // Hidden NavigationLink that activates when goToDetail == true
                NavigationLink(
                    destination: DetailView(),
                    isActive: $goToDetail
                ) {
                    EmptyView()  // invisible — triggered programmatically
                }
            }
            .navigationTitle("Home")
        }
    }
}

struct DetailView: View {
    var body: some View {
        Text("Navigated here automatically!")
            .font(.title)
            .navigationTitle("Detail")
    }
}

03-B

Programmatic Nav — NavigationPath

🔵 Level 2 — Intermediate

NavigationPath is the modern way to control the stack. path.append() pushes a view. path.removeLast() pops it.

ContentView.swift
import SwiftUI

struct City: Identifiable, Hashable {
    let id = UUID()
    let name: String
    let country: String
}

struct ContentView: View {
    @State private var path = NavigationPath()  // controls the stack

    let cities = [
        City(name: "New York",    country: "USA"),
        City(name: "Casablanca",  country: "Morocco"),
        City(name: "Tokyo",       country: "Japan")
    ]

    var body: some View {
        NavigationStack(path: $path) {
            VStack(spacing: 16) {
                Text("Stack depth: \(path.count)")
                    .font(.headline)
                    .foregroundColor(.secondary)

                ForEach(cities) { city in
                    Button("Push → \(city.name)") {
                        path.append(city)   // push programmatically
                    }
                    .buttonStyle(.borderedProminent())
                }

                if path.count > 0 {
                    Button("↩ Pop Last", role: .destructive) {
                        path.removeLast()   // pop one level
                    }
                    Button("🏠 Pop to Root") {
                        path.removeLast(path.count) // pop ALL
                    }
                }
            }
            .padding()
            .navigationTitle("Cities")
            .navigationDestination(for: City.self) { city in
                CityView(city: city, path: $path)
            }
        }
    }
}

struct CityView: View {
    let city: City
    @Binding var path: NavigationPath

    var body: some View {
        VStack(spacing: 20) {
            Text(city.name).font(.largeTitle).bold()
            Text(city.country).foregroundColor(.secondary)
            Button("🏠 Go Back to Root") {
                path.removeLast(path.count)
            }
            .buttonStyle(.borderedProminent())
        }
        .navigationTitle(city.name)
    }
}
03-C

Programmatic Nav — AppRouter Pattern

🟠 Level 3 — Complex

A shared AppRouter observable class owns the path, so any view in your app can navigate without passing bindings everywhere.

ContentView.swift
import SwiftUI

// ── Shared Router ──────────────────────────────────────
@Observable
class AppRouter {
    var path = NavigationPath()

    func push<T: Hashable>(_ value: T) { path.append(value) }
    func pop()           { if !path.isEmpty { path.removeLast() } }
    func popToRoot()     { path = NavigationPath() }
}

// ── Models ─────────────────────────────────────────────
struct Department: Identifiable, Hashable {
    let id = UUID()
    let name: String
    let icon: String
}
struct Employee: Identifiable, Hashable {
    let id = UUID()
    let name: String
    let role: String
}

// ── Root View ───────────────────────────────────────────
struct ContentView: View {
    @State private var router = AppRouter()

    let departments = [
        Department(name: "Engineering", icon: "hammer"),
        Department(name: "Design",      icon: "paintbrush"),
        Department(name: "Marketing",   icon: "megaphone")
    ]

    var body: some View {
        NavigationStack(path: $router.path) {
            List(departments) { dept in
                NavigationLink(value: dept) {
                    Label(dept.name, systemImage: dept.icon)
                }
            }
            .navigationTitle("Company")
            .toolbar {
                ToolbarItem(placement: .topBarTrailing) {
                    Button("Deep Link") {
                        // Jump 2 levels deep with one button press
                        let eng = Department(name: "Engineering", icon: "hammer")
                        let emp = Employee(name: "Khalid", role: "Lead Dev")
                        router.push(eng)
                        router.push(emp)
                    }
                }
            }

                        dismiss()
                    }
                    .disabled(title.isEmpty)
                }
            }

Alert, Popover & ConfirmationDialog — All Together

🟠 Level 3 — Complex

A real-world example combining alerts, confirmation dialogs (action sheets), and a popover in one view.

ContentView.swift
import SwiftUI

struct ContentView: View {
    @State private var showAlert        = false
    @State private var showConfirmDialog = false
    @State private var showPopover       = false
    @State private var statusLog: [String] = []

    var body: some View {
        NavigationStack {
            List {
                Section("Actions") {

                    Button("⚠️ Simple Alert") { showAlert = true }

                    // confirmationDialog = iOS action sheet
                    Button("🗑 Delete Account", role: .destructive) {
                        showConfirmDialog = true
                    }

                    Button("ℹ️ Show Popover") { showPopover = true }
                        .popover(isPresented: $showPopover) {
                            VStack(alignment: .leading, spacing: 12) {
                                Text("Sort Options").font(.headline)
                                Divider()
                                ForEach(["A → Z", "Newest", "Price ↑"], id: \.self) { opt in
                                    Button(opt) {
                                        statusLog.append("Sorted: \(opt)")
                                        showPopover = false
                                    }
                                    .frame(maxWidth: .infinity, alignment: .leading)
                                }
                            }
                            .padding()
                            .frame(minWidth: 200)
                        }
                }

                Section("Log") {
                    if statusLog.isEmpty {
                        Text("No actions yet").foregroundColor(.secondary)
                    }
                    ForEach(statusLog.reversed(), id: \.self) { entry in
                        Text(entry).font(.caption)
                    }
                }
            }
            .navigationTitle("Dialogs")
        }

        // Alert
        .alert("Are you sure?", isPresented: $showAlert) {
            Button("Yes")    { statusLog.append("Confirmed") }
            Button("No", role: .cancel) { statusLog.append("Cancelled") }
        } message: {
            Text("This will affect your data.")
        }

        // ConfirmationDialog (appears as action sheet on iPhone)
        .confirmationDialog(
            "Delete your account permanently?",
            isPresented: $showConfirmDialog,
            titleVisibility: .visible
        ) {
            Button("Delete Account", role: .destructive) {
                statusLog.append("Account deleted")
            }
            Button("Deactivate Instead") {
                statusLog.append("Account deactivated")
            }
            Button("Cancel", role: .cancel) {
                statusLog.append("Cancelled")
            }
        } message: {
            Text("You cannot undo this action.")
        }
    }
}
